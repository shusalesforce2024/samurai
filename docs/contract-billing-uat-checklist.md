# 契約請求・Freee連携 UATチェックリスト

更新日: 2026-05-24

## UATの目的
営業、経理、システム管理者それぞれの実務で、契約作成、初回請求、更新請求、Freee連携、入金ステータス同期、取消・再作成、レポート確認が問題なく使えることを確認する。

## 役割別の確認範囲
| 役割 | 主な確認範囲 |
| --- | --- |
| 営業 | 取引、見積、初回契約・請求作成、Freee取引先同期、Freee見積連携、Freee請求書作成、請求取消・再作成 |
| 経理 | 請求管理、請求明細確認、Freee請求書作成、取消・再作成、Freee入金ステータス確認、未入金・未請求の確認 |
| システム管理者 | 権限、タブ、ページレイアウト、Freee設定、商品マスタ、バッチスケジュール、エラー調査 |

## UAT前提条件
| No | 確認内容 | 結果 |
| --- | --- | --- |
| 1 | dev1に最新メタデータが反映されている |  |
| 2 | 営業ユーザーに `SAMURAI 営業` 権限セットが付与されている |  |
| 3 | 経理ユーザーに `SAMURAI 経理` 権限セットが付与されている |  |
| 4 | システム管理者に `SAMURAI システム管理者` 権限セットが付与されている |  |
| 5 | Freee設定が登録されている |  |
| 6 | 商品マスタが登録されている |  |
| 7 | テスト用取引先にFreee取引先IDがある、または同期できる |  |

## 営業UAT
| No | シナリオ | 操作 | 期待結果 | 結果 |
| --- | --- | --- | --- | --- |
| S-01 | Freee取引先同期 | 取引先画面でFreee取引先同期を実行する | Freee取引先IDがSalesforce取引先に保存される |  |
| S-02 | 見積作成 | 取引に見積・見積明細を作成する | 見積明細に商品マスタ、数量、単価、税率、値引き率が入る |  |
| S-03 | Freee見積連携 | 見積画面でFreee見積連携を実行する | Freee見積書ID、番号、URL、同期ステータスが保存される |  |
| S-04 | Activate見積採用 | 見積ステータスをActivateにする | 初回作成対象の見積として扱われる |  |
| S-05 | Activate見積重複エラー | 同一取引にActivate見積を2件作る | 初回作成時にエラーとなり手動確認できる |  |
| S-06 | 月契約の初回作成 | 月契約の取引から初回作成ボタンを押す | 契約、契約期間、請求、請求明細、契約月次明細が作成される |  |
| S-07 | 年契約の初回作成 | 年契約の取引から初回作成ボタンを押す | 年契約、12か月分の月次明細、年一括請求が作成される |  |
| S-08 | 複数商品 | Rendery年間契約とセキュリティオプションを含める | 年一括請求は商品名+年間利用料でまとまり、月次明細では商品別に追跡できる |  |
| S-09 | 初期費用 | 初回のみ商品を含める | 初期費用は初回請求だけに含まれ、更新請求には含まれない |  |
| S-10 | Freee請求書作成 | 請求・請求明細を確認後、Freee請求書作成を実行する | Freee請求書ID、番号、URL、送付ステータス、決済ステータスが保存される |  |
| S-11 | 請求取消 | 作成済み請求の取消を実行する | Salesforce請求が取消済みとなり、Freee側も取消扱いになる |  |
| S-12 | 請求再作成 | 取消済み請求から再作成する | 元請求を残したまま新規請求が作成され、元請求参照が保存される |  |

## UAT完了条件
- 営業の主要シナリオがOKである
- 経理の主要シナリオがOKである
- システム管理者の設定・バッチ確認がOKである
- 月契約、年契約、複数商品、初期費用、取消、再作成、入金同期の主要ケースがOKである
- NGが残る場合、リリース判定表で条件付き可または不可を判断している
