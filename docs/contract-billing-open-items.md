# 契約・請求・Freee連携 残課題

## 1. Freee側の仕様確認

業務要件は確定済み。
実装前に、Freee API側の仕様として以下を確認する。

```text
1. 請求書の取消・キャンセルAPI仕様
2. 請求書ステータス取得仕様
3. 送付・決済ステータス取得仕様
4. Freee品目IDの必須有無
5. 税額切り上げの指定可否
6. 請求書タイトル・明細名の反映項目
```

## 2. 確定済みとして反映済みの事項

以下は要件一覧、機能一覧、業務マニュアルへ反映済み。

```text
初回請求:
- 当月分から請求する
- 契約作成日が契約開始月より後の場合も、契約開始月分を請求する
- 月途中開始でも日割りなし

請求日・支払期日:
- 請求日は毎月20日
- 20日が土日の場合は前日
- 支払期日は翌月末

年払い請求日:
- 基本は利用開始月20日
- クライアントにより利用開始前月20日も可能
- 契約に年払い請求月区分を持つ

年契約の商品マスタ:
- 年間一括支払い分の商品マスタを用意する
- 年契約の月単位管理用の商品マスタを用意する
- 請求明細では年間一括支払い分を使用する
- 契約月次明細では月単位管理用を使用する

年契約の請求明細名:
- 契約名 + 年払い費用

アップセル:
- 基本的には翌月から適用する
- 旧契約を終了する
- 新契約を作成する
- 旧契約と新契約を関連付ける
- 作成済み請求は削除しない
- 決済待ち請求も削除せず残す
- 請求取消・再作成などの自動処理は行わない
- 必要な請求調整がある場合は、運用判断で手動対応する
- 日割りなし

Freee連携エラー:
- 画面上でエラーメッセージを表示する
- Freee連携ログオブジェクトにエラーを格納する
- 自動リトライ、手動リトライは行わない

権限・編集制御:
- 連携前: 編集可能
- 連携済み: 業務項目は編集不可、社内メモのみ編集可能
- 連携エラー: 修正に必要な項目のみ編集可能
- 取消済み: 編集不可、社内メモのみ編集可能

年契約の更新確認ステータス:
- 未確認、更新予定: 自動更新対象
- 更新停止、条件変更あり: 自動更新対象外
- 確認不要: 自動更新対象外

通知:
- 基本通知は行わない
- レポートまたはリストビューで確認する

バッチ実行時刻:
- 月契約の翌月分請求書作成: 毎月11日 2:00
- 年契約の次年度分請求書作成対象確認: 毎月11日 2:00
- Freee送付・決済ステータス同期: 毎日 2:00

契約月次明細レポート:
- 契約月次明細(営業)
- 契約月次明細(経理)
- 契約月次明細(解約)
```
