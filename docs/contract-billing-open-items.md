# 契約・請求・Freee連携 検討事項

## 1. 初回請求の対象期間

契約開始月を請求する方針だが、契約作成日が契約開始月より後の場合も開始月分でよいか確認が必要。
→当月分から請求する。

例:

```text
契約開始日: 2026/06/01
契約作成日: 2026/06/25
初回請求対象: 2026年6月分でよいか
```

関連する確定事項:

```text
月途中開始の場合は日割りなし
開始月を1か月分として請求する
```

## 2. 初回請求日・支払期日

初回請求書は契約作成時に即時作成する。
ただし、請求日・支払期日は毎月20日固定のため、契約作成時点で当月20日を過ぎている場合の扱いを確認する必要がある。
→支払期日は翌月末。請求日は毎月20日(土日の場合は前日)
→年払いの時は基本的には利用開始月の２０日だが、クライアントによっては前月の２０日になる場合がある。
推奨案:

```text
契約作成日以降で最初に来る20日を請求日・支払期日とする。
20日が土日の場合は前営業日とする。
当月分の20日が過去日であれば翌月20日とする。
```

確認例:

```text
契約作成日: 2026/06/25
請求日: 2026/07/20 でよいか
```

## 3. 年契約の年間利用料1明細の内訳追跡方法

年契約では、Freee請求書上は年間利用料1明細にまとめる。
ただし、Salesforce上では内訳を追跡したい。
→商品マスタに年間一括支払い分と年間の月単位のマスタを用意する

現時点の推奨:

```text
契約月次明細から商品別内訳を辿る。
請求明細単位で厳密な内訳管理が必要になった場合は、請求明細内訳オブジェクトを追加する。
```

関連案:

```text
商品マスタに年間一括請求用の商品と、年額を按分した月額料金マスタを持つ。

請求明細:
- 年間一括請求用の商品マスタを参照

契約月次明細:
- 月額料金マスタを参照
```

追加確認:

```text
複数商品を年契約1明細にまとめる場合、請求明細の商品マスタをどの商品にするか。
```

## 4. 年契約の請求明細名

年契約の請求明細名は「商品名 + 年間利用料」とする方針。
ただし、複数商品を1明細にまとめる場合、どの商品名を使うか確認が必要。
→契約名 + 年払い費用

候補:

```text
A. 主商品名 + 年間利用料
B. 契約名 + 年間利用料
C. 商品名を連結 + 年間利用料
D. 固定で「年間利用料」
```

推奨案:

```text
B. 契約名 + 年間利用料
```

## 5. アップセル時の契約切り替えルール

アップセルは発生する。
現状契約を切り替える対応とする方針だが、詳細ルールは未確定。
→基本的には翌月からアップセルとなる。推奨案でOK

確認事項:

```text
1. 旧契約は終了扱いにするか
2. 新契約を別レコードとして作成するか
3. 旧契約の契約期間・契約月次明細をどう扱うか
4. 作成済み請求をどう扱うか→削除する。
5. 未入金請求をどう扱うか→そのままにして、消さない
6. アップセル分をいつから請求対象にするか→翌月から
```

推奨案:

```text
旧契約を終了
新契約を作成
旧契約と新契約を関連付ける
作成済み請求は原則維持
変更により不要になった請求は取消
日割りなし
```

## 6. Freee側の仕様確認

Freee連携で確認が必要な事項:

```text
1. 請求書の取消・キャンセルAPI仕様
2. 請求書ステータス取得仕様
3. 入金ステータス取得仕様
4. Freee品目IDの必須有無
5. 税額切り上げの指定可否
6. 請求書タイトル・明細名の反映項目
```


## 8. 確定済み事項

以下は確認済みのため、要件一覧・機能一覧・業務マニュアルへ反映済み。

```text
Freee連携エラー時:
- 画面上でエラーメッセージを表示する
- Freee連携ログオブジェクトにエラーを格納する
- 自動リトライ、手動リトライは行わない

権限・編集制御:
- 連携前: 編集可能
- 連携済み: 業務項目は編集不可、社内メモのみ編集可能
- 連携エラー: 修正に必要な項目のみ編集可能
- 取消済み: 編集不可、社内メモのみ編集可能

年契約の更新確認ステータス:
- 未確認、更新予定: 自動更新対象
- 更新停止、条件変更あり: 自動更新対象外
- 確認不要: 自動更新対象外

通知:
- 基本通知は行わない
- レポートまたはリストビューで確認する

バッチ実行時刻:
- 月契約の翌月分請求書作成: 毎月11日 2:00
- 年契約の次年度分請求書作成対象確認: 毎月11日 2:00
- Freee入金ステータス同期: 毎日 2:00
```

## 9. 契約月次明細（レポート）
契約月次明細レポートは３つ必要となる。


```text
契約月次明細(営業)：MRRのグラフを表示する。すべての過去の金額が必要。ステータスが解約されたら、金額は変えずに解約レポートに表示させる。
契約月次明細(経理)：今有効な契約がわかればよい。過去２か月分入金されたもののみ表示する。
契約月次明細(解約)：本当は獲得できる予定だった契約とその金額。
```
